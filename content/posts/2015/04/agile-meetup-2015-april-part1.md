本次分享的講者是趨勢科技的[David Ko(柯仁傑)](https://www.facebook.com/kojenchieh)，他在2008年開始將Agile導入趨勢當中，並創立了[Scrum Community in Taiwan](https://www.facebook.com/groups/179345672472/)和[AgileCommunity.tw](https://www.facebook.com/AgileCommunity.tw)兩個社群，致力於在國內推廣Agile及Scrum。

本次主題是討論管理者如何建立一個具向心力、且高效能的開發團隊。畢竟Agile或Scrum都只是輔助軟體開發的流程，其運作核心還是在團隊成員之間的彼此相互配合。

首先，一個團隊的組成必須要有一位Leader或Manager，但這位Manager是透過太子/王儲制？高階主管會早在一年前告知某位員工，說明年何時要接主管職，這期間好好學習？不，通常都是無預警情況下的通知，讓人措手不及。

以軟體開發領域而言，這些被拔擢成為Manager的員工通常都是技術層面實力堅強，而絕非帶領團隊。這種情況中，不僅Manager無法扮演好自己的新角色，也非常容易影響與Developer的互動，造成彼此之間的不了解。

所以一個團隊的組成，是需要經過一段時間的觀察、嘗試以及訓練。以下是David分享的方法。

# 一. Find Right Person

在徵才面試當中，透過與應徵者的談話，了解其是否能培育成適合團隊的人才。

1. 考量條件 \
    創新、主動、學習能力及自我調適能力等。

2. 男女比例 \
    團隊成員中，男女比例的均衡對效率的影響。

3. 工作外興趣 \
    由專業領域以外的興趣是另一個能夠判斷求職者的面向。

# 二. 人才培育

1. 小組長輪替 \
    由團隊成員輪替小組長一職，一方面學習如何領導小團隊，同時也能避免固定同一人為小組長的壓力。

2. 工作輪調 \
    不要把成員固定放在某一件工作/職務上，應該採輪調的方式，每經過一段時間，輪調所有成員的工作，讓他們各自都能接觸、學習完成不同工作，同時可以藉此找出每個成員的擅長及興趣領域，未來可以針對該方向進行培育。

3. 打造專業形象 \
    在不斷進行工作輪調的同時，觀察成員對於技術或管理方面的才能，並針對其擅長領域加以培育，使其於該領域中展現專業形象。

4. 展示成果 \
    在適當的場合提供舞台，讓成員們有展現能力的空間，並且不吝於在眾人面前給予鼓勵及感謝。

5. 找人指導 \
    在工作任務外，尋找適合的講師來幫成員們上課，加強其能力。

# 三. 凝聚人心

1. 共同患難 \
    從少人數的小團隊先開始，例如R&D跟QA兩人，平常就開始一同進行工作，無論開發績效好壞或程式瑕疵皆是兩人賞罰與共。

2. 午餐約會 \
    主管與團隊成員們一同用餐，拉近彼此間的距離，使成員願意對其吐露心聲，但下屬對主管的心防並非一朝一夕就能突破。

3. 意外禮物 \
    出差或去國外旅遊時可以準備給每個成員的禮物，但不應該每件禮物都相同，而是每個人的都不一樣，以表示對每個成員各自不同的心意。

4. 玩在一起 \
    並非表示Manager需要親身下海跟成員們參與各種娛樂活動，而是指可以幫不同興趣的成員尋找相同志趣的伙伴。

# 四. 打造環境

1. 資訊透明化

2. 隨處可討論的機制 \
    爭取讓團隊成員能夠便於隨處討論的硬體設備，如雙螢幕、白板、板凳等等。

3. 後勤資源

# 總結

* 找對的人就像教小孩一樣 \
    不要只會念書，也要懂得過生活
* 培育人才就像教小孩一樣 \
    給空間去闖蕩，並讚美其成果
* 凝聚人心就像教小孩一樣 \
    花時間陪伴，才能建立信任感
* 打造環境就像教小孩一樣 \
    透明、對話、適當的金援

Meetup的尾聲，David分享了很棒的一句話
>『用人不在減少人的短處，而在發揮其長處』。

# Q&A

Q: 工作輪調後到如果在新職務上發揮的相當好，那麼是否還需要再繼續輪調？ \
Ans: 可以，因為不知道繼續輪調後會不會更好的發揮。

Q: 工作輪調後難易度是否影響考績評鑑？ \
Ans: 工作效率要跟職等相同。

Q: 如何引導成員朝向技術或管理專精？ \
Ans: 需要兩到三年的時間讓成員了解自己的擅長及興趣。
